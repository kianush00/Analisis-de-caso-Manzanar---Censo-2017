Introducción¶

Propósito del análisis¶

El propsito de este desarrollo consiste en resolver la incertidumbre que se genera cada vez que se levantan los datos del CENSO, ya que, a veces, no es posible acceder a revisar el tipo de terreno o tipo de entidad, siendo estas estimadas o simplemente indeterminadas, ya sea por fallas de los censistas, falta de permisos o falta de disposicin de los residentes. De esta manera, los datos para futuros estudios quedan inconclusos y/o con vacos en la informacin, generando trabajo extra para los analistas de datos que deseen trabajar con esa informacin para realizar otros tipos de estudios.

Para solucionar este problema, se plantean mtodos para predecir la clase de la manzana, dgase el tipo de entidad (campo, pueblo, casero, etc.) que esta representa, junto con su clasificacin urbana o rural, con el fin de poder completar los datos para futuros anlisis.

A base de los datos obtenidos por el censo, entre los cuales se encuentra el acceso a agua potable y la materialidad de las casas dentro de la manzana, podra ser posible obtener tal informacin.

Cambio de enfoque¶

Tomando en cuenta la retroalimentacin de los avances anteriores, se decidi abandonar la idea de predecir algn tipo de indicador de pobreza con los datos del censo. En vez de eso, se tom una alternativa ms realista, la cual consiste en responder preguntas relacionadas con la materialidad de las manzanas y la categorizacin de clases indeterminadas a base de atributos de materialidad, agua y otros sobre los que se realizarn pruebas para determinar cules son los ms apropiados para realizar predicciones y responder a las interrogantes.

Índice de materialidad de la manzana¶

Se genera un nuevo dato que corresponde al ndice de materialidad de la manzana, el cual entrega una idea general de la calidad promedio de los materiales de la manzana, teniendo en cuenta que las viviendas se pueden clasificar por materialidad aceptable, recuperable e irrecuperable. Para calcular este ndice se implementa la siguiente frmula:

$ indice.mat = \frac{(MAT.ACEPTABLE + MAT.RECUPERABLE + MATIRREC)}{(MAT.ACEPTABLE + 2 * MAT.RECUPERABLE + 4 * MATIRREC)} $

Este ndice se encuentra dentro del rango [0.25, 1], es decir, a mayor valor, la manzana posee una mejor calidad de materialidad en promedio y viceversa.

Problemas¶

Problema 1¶

El primer problema consiste en predecir la clase NOM_CAT_ENT de una manzana, la cual corresponde a su categora de entidad (ciudad, pueblo, aldea, casero, etc.). Lo anterior se eligi luego de hacer el anlisis exploratorio y observar que muchas filas tienen la columna NOM_CAT_ENT con valor indeterminado. Este problema se considera como clasificacin multiclase.

Problema 2¶

El segundo problema consiste en predecir el atributo NOM_AREA que corresponde al tipo de rea de una manzana, este atributo indica si la manzana se encuentra en un rea rural o urbana. Este problema se identifica como clasificacin binaria.

Método de buscar clasificador¶

Se abordarn los dos problemas propuestos de manera similar. En el caso del problema de clasificacin de la columna NOM_CAT_ENT se crear un modelo de prediccin. Para hacer el modelo se entrenar un rbol de decisin. Se elige este tipo de predictor debido a que se infiere que una categora de rea para una manzana se podra decidir en base a decisiones binarias.

El rbol se entrenar en base a distintas configuraciones, es decir, utilizar un rango de 1 a 14 profundidades mximas, junto con 2 criterios que corresponden al de entropa y gini, para comprobar cul es la combinacin de parmetros ms adecuada para predecir NOM_CAT_ENT.

Para entrenar los rboles de decisin, se separar el dataset mencionado en un conjunto de entrenamiento y otro de prueba, los cuales se dividirn en la proporcin 70/30 respectivamente, ya que se considera que al utilizar el doble de datos para entrenamiento, se obtendr un modelo capaz de predecir el tercio restante.

def getBestDecisionTreeParameters(X_train, X_test, y_train, y_test):
    criterios = ["entropy", "gini"];   #se evaluar la pureza de los nodos en base a los criterios de entropa y gini
    max_depths = range(1,15);  #se utilizar un rango de 1 a 14 profundidades mximas

    best_precision = 0;
    best_criterio = "";
    best_depth = 0;
    best_tree_clf = DecisionTreeClassifier();

    for c in criterios:
        for d in max_depths:
            clf_orig = DecisionTreeClassifier(criterion=c,max_depth=d);
            clf_orig.fit(X_train, y_train);
            y_pred = clf_orig.predict(X_test);
            #el promedio de precisiones se obtiene calculando esta mtrica de forma global, contando el total de TP, FN y FP
            precision = precision_score(y_test, y_pred, average='micro'); 
            if precision > best_precision:
                best_precision = precision;
                best_criterio = c;
                best_depth = d;
                best_tree_clf = clf_orig;
            print("Precisin con criterio: [" + str(c) + "] profundidad [" + str(d) + "]:  " + str(precision));

    y_pred_tree = best_tree_clf.predict(X_test);
    return best_precision, best_criterio, best_depth, best_tree_clf, y_pred_tree;

def getBestKNNParameters(X_train, X_test, y_train, y_test):
    algorithms = ["brute", "kd_tree", "ball_tree"];   #se evalan los algoritmos de fuerza bruta, k-d tree y ball tree
    n_neighbors = [14, 18, 24, 32];  #se probar un rango de vecinos que va de 14 a 32, eligiendo valores intermedios

    best_precision = 0;
    best_algo = "";
    best_n = 0;
    best_knn_clf = KNeighborsClassifier();

    for c in algorithms:
        for d in n_neighbors:
            clf_orig = KNeighborsClassifier(algorithm=c,n_neighbors=d);
            clf_orig.fit(X_train, y_train);
            y_pred = clf_orig.predict(X_test);
            #el promedio de precisiones se obtiene calculando esta mtrica de forma global, contando el total de TP, FN y FP
            precision = precision_score(y_test, y_pred, average='micro');
            if precision > best_precision:
                best_precision = precision;
                best_algo = c;
                best_n = d;
                best_knn_clf = clf_orig;
            print("Precisin con algoritmo: [" + str(c) + "] n_vecinos [" + str(d) + "]:  " + str(precision));

    y_pred_knn = best_knn_clf.predict(X_test);
    return best_precision, best_algo, best_n, best_knn_clf, y_pred_knn;

Análisis de resultados¶

Análisis problema 1¶

El resultado obtenido no se considera bueno, pues no resuelve el problema de llenar los atributos indeterminados, ya que se consider que alrededor de un 50% de datos fallidos en un conjunto de datos oficial del gobierno no es aceptable. Este resultado tiene sentido ya que, como se mencion anteriormente, la mayora de las manzanas posee una categora indeterminada, lo que, junto con la baja correlacin entre los atributos utilizados para predecir y la clase, se produjo un alto sesgo en los resultados.

Se observa que el rendimiento del rbol de decisin es superior al modelo KNN en este problema, esto se debe a que se encuentra ms optimizado el modelo de rbol para problemas multivariable y debido a la naturaleza misma de los datos del problema, la clusterizacin de los datos no se encuentra tan definida, adems de que KNN por lo general implica un costo computacional ms elevado.

En base a las categoras que mejor se predicen (Aldea, Comunidad Indgena y Parcela-Hijuela), se entiende que este comportamiento se debe a la cantidad de datos que contenan esta clasificacin, adems que comparten el hecho comn de contar con agua de origen rural en gran medida, lo cual es un hecho significativo para la prediccin. Por otro lado, entre las categoras que peor se predicen (Asentamiento Minero, Asentamiento Pesquero, Campamento y Otros) tienen en comn que son muy pocas manzanas las que tenan esas categoras en comparacin con el resto, lo que produjo precisiones muy bajas para estas 4 categoras, llegando incluso a tener 0% de precisin en ciertos casos.

Análisis problema 2¶

La variacin entre el peor y mejor modelo es pequea, pero ya que los modelos obtenan una precisin promedio tan alta se considera que la metodologa efectivamente ayud a elegir un mejor modelo.

El resultado obtenido se considera bueno ya que su precisin es superior a 90%, esto indica que si se pueden utilizar los atributos del censo para predecir la clase NOM_AREA debido a que, segn la mtrica elegida, 9 de cada 10 predicciones sern correctas y es posible mitigar el 10% de prdida con otras tcnicas a futuro.

En este problema se denota que el entrenamiento de KNN es ms costoso ya que es un problema multivariable con una salida binaria, es difcil clasificar de forma binaria con KNN ya que el clustering se vuelve muy complejo. Al igual que en el primer problema, el rbol de decisin posee un rendimiento mayor que KNN.

Conclusión¶

Para concluir este experimento, se denota que la intencin de este era completar los datos de la plataforma CENSO para mejorar la calidad de los estudios basados en este, por lo tanto se realiz una propuesta experimental que contemplaba el uso de los atributos disponibles para la prediccin de la categora de la manzana, junto con la prediccin de su rea urbana o rural, siendo estos problemas de clasificacin como fue dicho anteriormente. Luego de realizar el experimento, en el primer problema se obtuvo un resultado deplorable de 50% desde la mtrica de precisin, por lo tanto, para este problema se concluye que los atributos no presentaban una relacin fuerte con la clase, algo que se poda percibir ms no asegurar en la matriz de correlacin. Sin embargo, en el segundo problema se obtuvo un resultado satisfactorio, con un 90% de precisin, lo que permite utilizar modelo de tal problema para predecir esa categora en base a variables como el origen del agua, la calidad de materialidad y el tipo de vivienda.

En base a lo anterior, se concluye que el estudio fue significativo para modelar el problema y plantearlo para futuros estudios con otras variables que permitan refinar el resultado. Por otra parte, se obtuvo un significativo resultado en el problema de clasificacin de categoria urbana o rural, ya que este contaba con datos consistentes y en la totalidad del dataframe, siendo exitoso en resolver el problema y til para su utilizacin en futuros estudios.
